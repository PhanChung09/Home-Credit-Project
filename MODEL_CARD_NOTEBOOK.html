Executive Summary (One-Page)

Business Recommendation

Deploy the tuned XGBoost model from Modeling.qmd with a decision threshold selected by expected-value analysis (recommended operating range around 0.35 probability of default, to be validated with current portfolio economics).

Expected Financial Impact

At the selected threshold, the business can reduce expected credit losses by rejecting more likely defaulters while still approving the majority of low-risk applicants. The threshold optimization section quantifies expected value under explicit assumptions for: - profit on repaid loans, - loss given default, - recovery rates.

Key Caveats

  • Class imbalance (~8% defaults) makes accuracy misleading; AUC/precision/recall are prioritized.
  • Model depends on quality and availability of external credit features (EXT_SOURCE_*).
  • Fairness monitoring is required due to potential disparities by gender and education.
  • Performance can degrade over time (economic drift), so periodic retraining and monitoring are required.

3) Performance Metrics

We report AUC plus classification metrics (precision/recall) at the chosen threshold.

Code 
library(caret)
library(pROC)
library(xgboost)

set.seed(42)

# Basic train matrix prep
prep_for_model <- function(df) {
  out <- df
  drop_cols <- intersect(c("SK_ID_CURR"), names(out))
  out <- out %>% select(-all_of(drop_cols))

  y <- out$TARGET
  X <- out %>% select(-TARGET)

  # Encode non-numeric
  for (nm in names(X)) {
    if (!is.numeric(X[[nm]])) X[[nm]] <- as.numeric(as.factor(X[[nm]]))
  }
  X[is.na(X)] <- -999
  list(X = X, y = y)
}

pp <- prep_for_model(train)
X <- pp$X
y <- pp$y

# 3-fold CV AUC estimate
folds <- createFolds(y, k = 3, list = TRUE, returnTrain = FALSE)
auc_vec <- c()

for (f in seq_along(folds)) {
  te <- folds[[f]]
  tr <- setdiff(seq_len(nrow(X)), te)

  dtr <- xgb.DMatrix(as.matrix(X[tr, ]), label = y[tr])
  dte <- xgb.DMatrix(as.matrix(X[te, ]))

  fit <- xgb.train(params = best_params, data = dtr, nrounds = 200, verbose = 0)
  p <- predict(fit, dte)
  auc_vec <- c(auc_vec, as.numeric(auc(y[te], p)))
}

cv_auc_mean <- mean(auc_vec)
cv_auc_sd <- sd(auc_vec)

# Fit once on full train to compute precision/recall at chosen threshold
dall <- xgb.DMatrix(as.matrix(X), label = y)
fit_all <- xgb.train(params = best_params, data = dall, nrounds = 200, verbose = 0)
prob_all <- predict(fit_all, dall)

chosen_threshold <- 0.35
pred_all <- ifelse(prob_all >= chosen_threshold, 1, 0)

precision <- ifelse(sum(pred_all == 1) > 0, sum(pred_all == 1 & y == 1) / sum(pred_all == 1), NA)
recall <- sum(pred_all == 1 & y == 1) / sum(y == 1)

perf_tbl <- tibble(
  Metric = c("CV AUC (mean)", "CV AUC (sd)", "Precision @0.35", "Recall @0.35", "Kaggle Public AUC"),
  Value = c(round(cv_auc_mean, 4), round(cv_auc_sd, 4), round(precision, 4), round(recall, 4), 0.7940)
)

4) Decision Threshold Analysis

We optimize threshold using expected value with realistic lending economics.

4.1 Cost Assumptions and Sources

Assumptions used for sensitivity analysis: - Profit on repaid loan: +$1,500 - Loss given default (LGD): $12,000 (assuming ~80% LGD) - Recovery rate: ~20%

Reference context: - Basel Committee on Banking Supervision (Basel framework, LGD concepts for unsecured retail exposure) - World Bank / IFC lending risk literature on emerging market retail recovery variability - Public consumer lending economics disclosures (portfolio-level default/loss dynamics)

These assumptions are scenario inputs for threshold optimization, and should be calibrated using institution-specific portfolio P&L.

4.2 Sensitivity Analysis Across Thresholds

Code 
threshold_grid <- seq(0.05, 0.80, by = 0.05)

profit_repaid <- 1500
loss_default <- 12000

threshold_results <- lapply(threshold_grid, function(t) {
  pred <- ifelse(prob_all >= t, 1, 0)  # 1 = predict default (deny)

  # Confusion-style counts where positive class is default
  TP <- sum(pred == 1 & y == 1) # correctly deny default
  FP <- sum(pred == 1 & y == 0) # deny good applicant
  TN <- sum(pred == 0 & y == 0) # approve good applicant
  FN <- sum(pred == 0 & y == 1) # approve default

  # Expected value approximation
  # approve good => +profit_repaid
  # approve default => -loss_default
  # deny good/default => 0 immediate P&L in this simple framing
  ev <- TN * profit_repaid - FN * loss_default

  tibble(
    threshold = t,
    precision = ifelse((TP + FP) > 0, TP / (TP + FP), NA_real_),
    recall = TP / (TP + FN),
    approval_rate = mean(pred == 0),
    expected_value = ev
  )
}) %>% bind_rows()

best_row <- threshold_results %>% arrange(desc(expected_value)) %>% slice(1)
threshold_results %>% arrange(desc(expected_value)) %>% head(10)
